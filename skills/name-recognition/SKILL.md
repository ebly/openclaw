# 机器人名字识别

## 核心原则

**只在被呼叫时响应** - 如果消息不是直接针对你，保持沉默（NO_REPLY）。

## 识别逻辑

### 1. 检查消息内容

判断消息是否包含以下内容：
- 你的名字（IDENTITY.md 中的 name）
- 你的昵称或称呼
- 明确指向你的关键词（如"美美"、"小刘"等）

### 2. 判断呼叫意图

以下情况视为**被呼叫**：
- ✅ 消息中直接提到你的名字
- ✅ 消息中使用"@"提到你的昵称
- ✅ 消息中明确说"XX（你的名字），你能...吗"
- ✅ 消息中包含针对你的问题或请求

以下情况视为**未被呼叫**：
- ❌ 消息中提及其他机器人的名字
- ❌ 消息是群聊中的通用讨论
- ❌ 消息与其他机器人相关
- ❌ 消息与自己无关

### 3. 响应规则

**如果被呼叫：**
- 正常回复消息
- 提供帮助或回答问题

**如果未被呼叫：**
- 回复 `NO_REPLY`
- 保持沉默，不发送任何消息

## 实现示例

### 示例 1: 识别自己的名字

```python
# 你的名字是"美美"
message = "美美，你能帮我设计个logo吗？"

if "美美" in message:
    # 被呼叫了，正常响应
    respond("好的！我来帮你设计logo~ 🎨")
else:
    # 未被呼叫，保持沉默
    NO_REPLY
```

### 示例 2: 检查是否被呼叫

```python
# 你的名字是"小刘"
message = "老张，你能@到小刘吗"

if "小刘" in message and "你" in message and any(word in message for word in ["能", "能吗", "可以吗"]):
    # 消息中提到你，但不是在直接叫你，而是在讨论你
    NO_REPLY
elif "小刘" in message and ("你" in message and ("在吗" in message or "来" in message)):
    # 消息是在叫你
    respond("我在！有什么能帮你的？")
else:
    NO_REPLY
```

### 示例 3: 避免误响应

```python
# 你的名字是"美美"
message = "小二，记得以后每天早上8点提醒我"

# 消息中没有提到"美美"，所以不应该响应
NO_REPLY
```

## 常见场景

### 场景 1: 用户呼叫其他机器人

**消息：** "小刘在吗"
**判断：** 如果你是"美美"，这个消息不是在叫你
**响应：** NO_REPLY

### 场景 2: 用户呼叫你

**消息：** "美美，能帮我设计个图吗？"
**判断：** 消息中明确提到"美美"，是在叫你
**响应：** 正常回复，提供帮助

### 场景 3: 用户在讨论但未呼叫

**消息：** "老张，小刘是机器人，你怎么能让自己看到他"
**判断：** 如果你是"小刘"，消息提到你但不是在叫你
**响应：** NO_REPLY

### 场景 4: 群聊通用讨论

**消息：** "大家觉得这个想法怎么样？"
**判断：** 通用讨论，没有呼叫特定机器人
**响应：** NO_REPLY

## 高级判断

### 复杂呼叫模式

有些消息需要更复杂的判断：

```python
# 你的名字是"小刘"
message = "老张，你能@到小刘吗"

# 分析消息：
# 1. 提到了"小刘"
# 2. 但说话的对象是"老张"
# 3. 内容是"能否@到小刘"，不是在叫小刘
# 结论：不是在叫你
NO_REPLY
```

```python
# 你的名字是"小刘"
message = "小刘在吗"

# 分析消息：
# 1. 直接提到"小刘"
# 2. 问题是"在吗"，是直接的呼叫
# 结论：是在叫你
respond("我在！有什么需要帮助的吗？")
```

## 配置你的名字

在 `IDENTITY.md` 中设置你的名字：

```markdown
- **Name:** 美美
- **What to call them:** 老大
```

这个技能会自动读取你的名字来判断是否被呼叫。

## 注意事项

1. **大小写不敏感** - "美美"和"美美"应该都能识别
2. **包含关系** - 如果你的名字包含在另一个名字中（比如"小刘"和"小李"），需要精确匹配
3. **避免误判** - 如果不确定是否被呼叫，倾向于保持沉默
4. **群聊语境** - 在群聊中要更谨慎，避免误响应
5. **学习用户习惯** - 观察用户如何称呼你，调整识别逻辑

## NO_REPLY 的使用

`NO_REPLY` 是特殊的响应，表示：
- 不发送任何消息到群里
- 完全保持沉默
- 不打扰其他机器人或用户

使用场景：
- 未被呼叫时
- 消息与自己无关时
- 其他机器人正在被呼叫时

## 技能集成

这个技能应该被加载到需要名字识别的所有机器人中：

1. 在每次收到群聊消息时自动触发
2. 在响应前先判断是否被呼叫
3. 如果未被呼叫，立即返回 NO_REPLY

这样可以确保机器人只在被呼叫时响应，提高交流效率和准确性。
